Sciter：react中子组件通知父组件的三种方法

2021-12-17

通过属性传递回调函数

传统的解决方案，通过在子组件上定义类似onXXX的属性来传递一个回调函数给子组件。
注意重点需要bind函数上下文，否则回调函数中this没有指向当前组件实例。

export class Root extends Element {
    constructor(props) {
        super();
        this.doHandle= this.doHandle.bind(this); // 注意要绑定函数上下文
    }

    doHandle() {
        console.log('doHandle');
    }

    render() {
        return (
            <div>
                <Child onFoo={this.doHandle}></Child>
            </div>
        )
    }
}

子组件中这样调用即可

this.props.onFoo();

冒泡事件

通过Element的dispatchEvent或PostEvent发送事件（dispatchEvent意味着同步事件，PostEvent意味着异步事件）。
在子组件中产生一个冒泡事件

this.postEvent(new Event('myevent', { bubbles: true }));

父组件中捕获事件即可

['on myevent'](evt, ele) {
    console.log('myevent');
}

https://github.com/c-smile/sciter-js-sdk/tree/main/docs/md/Element.md

全局事件

sciter特有的特性，进程级别的事件，所有窗口都会收到事件。
发送全局事件：Window.post(new Event('event-name'));
订阅全局事件：Element.onGlobalEvent('event-name', callback);
https://github.com/c-smile/sciter-js-sdk/tree/main/samples.sciter/global-events
